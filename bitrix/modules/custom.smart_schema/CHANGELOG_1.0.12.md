# 1.0.12

Качество разметки и производительность вывода.

1. **Списки (`ItemList`) без мусора.** Извлечение элементов списка (`genericListItems`) переведено на DOM и теперь отбрасывает навигацию: ссылки из шапки, меню, сайдбара и подвала (`<header>/<nav>/<footer>/<aside>`, классы `menu/nav/breadcrumb/sidebar/...`), а также `tel:`/`mailto:`/`javascript:` и пустые якоря. В `ItemList` попадают только ссылки из основного контента страницы.

2. **Отзывы строже.** `extractReviews` больше не принимает произвольный текстовый блок за отзыв (раньше так туда мог попасть, например, ответ из FAQ). Теперь элемент считается отзывом только при наличии автора, оценки или явной разметки `review` (`itemprop`/`itemtype`). Добавлено определение автора по `itemprop="author"` и пропуск контейнеров-обёрток, чтобы один отзыв не дублировался вместе со своим блоком-списком.

3. **Не запускать полный разбор HTML на каждой странице.** Дорогой анализ HTML (DOM + десятки регулярок по всей странице) в `Output::inject()` теперь выполняется только когда для URL действительно есть «страничные» предложения (точный URL или подходящий `page_kind`). Если для страницы нужен лишь sitewide-блок (Organization/WebSite из готового JSON) — тяжёлый анализ пропускается. Проверка наличия делается одним лёгким SQL-запросом (`Db::hasRequestProposals`).

4. **Нет «висящих» ссылок `@id`.** Если страничная разметка (`WebPage`/`Product`/…) ссылается на `#organization`/`#website`, а эти узлы не подтверждены как sitewide и не найдены в разметке страницы, модуль автоматически добавляет компактные `Organization`/`WebSite` из настроек, чтобы ссылки `@id` разрешались. Дубли не создаются: если узел уже есть в выводе или на странице, он не добавляется.

Обновление обратно совместимо, схема БД не меняется. После обновления очистите кеш Битрикса.
