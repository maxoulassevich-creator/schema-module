# 1.0.13

## Мульти-тип Product + Book для книг
- Карточки-книги теперь размечаются как `"@type": ["Product", "Book"]`. Товарные поля (`offers`, `brand`, `availability`, `sku`) остаются без изменений — Яндекс Товары / Яндекс Бизнес видят полноценный `Product`, — а сверху добавляется книжный смысловой слой: `author`, `isbn`, `bookFormat`, `inLanguage`, `numberOfPages`, `publisher`.
- Book включается только если это действительно книга — найден `автор` или `ISBN`. Обычные товары (канцтовары, цифровые товары) остаются чистым `Product` без книжных полей.
- Данные берутся из реальной страницы: свойства карточки Аспро (Автор/Издательство/Язык/ISBN/Переплёт/Количество страниц), существующий JSON-LD `Book` и текст. Модуль ничего не выдумывает.
- ISBN нормализуется (10/13 цифр); ISBN-13 дополнительно кладётся в `gtin13` (это же GTIN-13) — полезно для товарных сниппетов. Язык нормализуется в код (`английский → en`), тип обложки — в значения `schema.org` (`Hardcover`/`Paperback`/`EBook`/`AudiobookFormat`).

## Массовый откат всех внедрений
- В админке (вкладки «Обзор» и «Что внести») появилась кнопка **«Откатить все внедрения»** — снимает вывод разметки со всех подтверждённых пунктов сразу, включая оставшиеся от прошлых версий модуля. Данные и журнал при этом сохраняются, пункты можно внести заново.

## Сохранность внедрений между переустановками
- Уточнена формулировка при удалении модуля: галочка «Сохранить таблицы и настройки» включена по умолчанию и снабжена пояснением. Если её не снимать, все подтверждённые внедрения, история и настройки переживают удаление модуля и снова доступны после установки новой версии.
- В «Обзоре» добавлено пояснение: разметка выводится динамически, поэтому откат/удаление мгновенно убирают её со всех страниц (файлы шаблона и товары не меняются), а состояние хранится в БД.

## Исправление: порча кириллицы в значениях
- Исправлена обрезка значений в `normalizeItemName()`: обычный `trim()` резал строку по байтам, из-за чего хвостовые кириллические буквы, чей последний байт совпадал с байтами символов `·›»`, портились (например, «Автор» → «Авто�», «Язык» → «Язы�»). Обрезка переведена на многобайтно-безопасный вариант. Затрагивало имена в хлебных крошках, элементах списков и свойствах товара.

После обновления до 1.0.13 очистите кеш Битрикса и запустите анализ заново.
