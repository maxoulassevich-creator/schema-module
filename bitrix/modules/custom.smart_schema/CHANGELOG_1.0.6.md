# custom.smart_schema 1.0.6

Добавлена проверка карточек товаров по требованиям Яндекс Товаров к семантической разметке Product.

Что изменено:

- Сканер теперь отдельно проверяет карточки товара на наличие Product/ProductGroup в JSON-LD и microdata.
- Для Product проверяются обязательные для Яндекс Товаров поля: name, brand, image, offers.
- Для Offer/AggregateOffer проверяются price или lowPrice, priceCurrency, availability.
- Если существующая Product-разметка есть, но не хватает обязательных полей, модуль создаёт пункт замены Product во вкладке «Что внести».
- При построении Product модуль нормализует availability вида InStock в https://schema.org/InStock.
- Добавлен fallback для brand: можно подставлять название организации из настроек, если бренд/производитель/издательство не найдены на странице.
- Добавлена настройка включения/отключения проверки Product по правилам Яндекс Товаров.
