# 1.0.14

Доработка книжной карточки `Product` + `Book` по реальным страницам Аспро Премьер (проверено на карточке книги RELOD).

## Извлечение книжных данных
- Книжные поля теперь читаются в том числе из `additionalProperty` существующего `Product` JSON-LD (в Аспро там лежат Автор/Язык/Формат/Год издания/Возраст), а не только из DOM-свойств карточки. Это самый надёжный источник.
- Исправлен матч заголовка свойства «Формат» (раньше ловилось только «Формат издания»/«Переплёт»), поэтому значение «Мягкая обложка» корректно даёт `bookFormat: Paperback`.
- Добавлены поля `datePublished` (из «Год издания») и `typicalAgeRange` (из «Рекомендуемый возраст»).
- `isbn` берётся из `SKU`, когда он выглядит как ISBN-13/10 (у книг артикул = ISBN). ISBN-13 дублируется в `gtin13`.

## Бренд книги — без магазина
- Для книг магазин больше не подставляется в `brand`/`publisher`. Используется только реальный издатель/бренд, найденный на странице; если его нет — поля просто не выводятся (лучше предупреждение Яндекса, чем неверные данные). Чтобы бренд заполнялся автоматически по всему каталогу, выведите свойство «Издательство» в характеристики товара — модуль его подхватит.
- Для НЕ-книг прежнее поведение сохранено: при пустом бренде работает fallback на название организации (настройка из 1.0.7).

## Рейтинг
- Модуль по-прежнему выводит рейтинг только из реально найденных отзывов и не переносит непроверяемый `aggregateRating`. Это важно, потому что встречаются карточки с разметкой рейтинга (`ratingValue`), которого нет на странице («Нет отзывов») — такой рейтинг является нарушением правил Google, и режим «Заменить своей» корректно его убирает.

После обновления до 1.0.14 очистите кеш Битрикса и запустите анализ заново.
