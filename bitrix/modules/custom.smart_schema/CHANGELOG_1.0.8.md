# 1.0.8

- Расширена классификация ручных URL: контакты, о компании, доставка, оплата, возврат/гарантия, юридические страницы, FAQ, отзывы, поиск, бренды/издательства, услуги, мероприятия, видео, корзина, оформление заказа, профиль и обычные информационные страницы.
- Ручной URL теперь сначала анализируется по уже найденной JSON-LD/microdata-разметке, затем по URL, H1/title/meta description и тексту полного HTML.
- Добавлены корректные Schema.org-типы для новых классов страниц: AboutPage, ContactPage, SearchResultsPage, ProfilePage, Service, Event, CollectionPage, ItemList и WebPage fallback.
- Старые pending-предложения для ручного URL с менее точной классификацией автоматически помечаются как «не вносить» после повторного анализа.
- Усилена защита от ложных Product/Offer на информационных страницах: товарная разметка предлагается только для настоящих товарных страниц или уже найденного Product/ProductGroup.
- Для неизвестных страниц используется безопасный fallback WebPage; модуль не выдумывает специальные типы без подтверждения в контенте или коде страницы.
