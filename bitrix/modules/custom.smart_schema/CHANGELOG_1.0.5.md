# custom.smart_schema 1.0.5

Исправления проверки после внесения/замены:

- Проверка теперь запрашивает страницу с одноразовым cache-busting параметром, чтобы не получать старую HTML-версию из кеша сразу после применения.
- HTTP-запрос проверки отправляет заголовки `Cache-Control: no-cache` и `Pragma: no-cache`.
- Поиск JSON-LD-блока модуля стал устойчивее к порядку HTML-атрибутов и смене ID предложения после повторного анализа.
- В журнал проверки добавлен `verification_url`, `microdata_types` и счётчик отключённой microdata.
- Сообщение проверки точнее различает ситуации: блок модуля не найден, блок найден, но старая microdata осталась, или разметка найдена, но не от модуля.
